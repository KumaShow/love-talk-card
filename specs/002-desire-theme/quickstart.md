## 3. 內容安全審查清單（每張 desire 卡逐項）— 對應 FR-006 / US4 / SC-005

- [ ] 成人但**非露骨技巧教學**；鼓勵描述偏好/界線/慢慢探索，而非要求立即行動（AS US4-1）。
- [ ] 親密頻率題允許差異與協商，**不暗示任一方有義務**滿足對方（US4-2）。
- [ ] 邀請方式題包含**可拒絕、可暫停、可改天**的語氣（US4-3）。
- [ ] **不含**未成年、非自願、脅迫、酒醉失去判斷、羞辱貶低、暴力傷害、露骨技巧教學、醫療性建議（FR-006 / US4-4）。
- [ ] 語氣尊重、可拒絕、以雙方同意與安全為核心；非測驗式評分、非強迫揭露。
- [ ] 翻譯（th）保留「成人、同意、安全、可拒絕」語氣，不因語言轉換更露骨或更含糊（Edge Case）。

## 4. 主題邊界分類清單（候選題目 → 主題）— 對應 FR-010 / US3 / SC-002

本節為 `002-desire-theme` 的**權威分類來源**。當候選題目在 `attraction` / `self` / `interaction` / `trust` / `desire` 之間需要擇一時，應先依本節的「主要對話任務」規則判斷，再回看 spec 的 CB-001～CB-006 與 US3 acceptance scenarios；若與其他說明文件出現表述差異，以本節的分類清單與判斷要訣為準。

依**題目的主要對話任務**判斷，落點唯一：

| 若主要任務是… | 歸入 | 依據 |
|----------------|------|------|
| 火花、心動、魅力、被吸引、浪漫注目 | `attraction` | CB-001 / US3-1 |
| 自我覺察、個人需求、情緒模式、對慾望/界線的**內省** | `self` | CB-002 / US3-2 |
| 日常習慣、溝通風格、玩笑、衝突模式、相處節奏 | `interaction` | CB-003 |
| 可靠、情感安全、脆弱、修復、承諾 | `trust` | CB-004 / US3-4 |
| **成人身體親密、性需求、碰觸偏好、親密頻率、邀請、同意協商** | `desire` | CB-004 例外 / US3-3 |

**判斷要訣**：
- 重點是「**伴侶間身體協商**」→ desire；重點是「**自我理解**」→ self。
- 信任/安全題若主體是「**成人身體親密中的同意與界線**」→ desire；否則留 trust。
- 既有四主題 intimate 卡疑似越界（主要任務變成成人身體親密協商）→ 移至 desire／改寫回原尺度／暫緩（CB-006 / US2-5）。

**Acceptance scenario 對照**：
- US3-1：心動、吸引、浪漫注目 → `attraction`
- US3-2：理解自己的慾望、界線、害羞與需求 → `self`
- US3-3：邀請身體親密、碰觸偏好、同意協商 → `desire`
- US3-4：受傷後的修復、安全感、信任重建 → `trust`
- 補充：日常互動節奏、玩笑、溝通與衝突習慣 → `interaction`

## 5. 完成判準（對照 Success Criteria）

- SC-001 第 5 主題不需 one-off pipeline、不需 15+5 分層 → 由型別/資料加法達成。
- SC-006/SC-007 進入前可辨識為成人主題、可選可避 → 由分層提示 + 攔截式 notice 達成。

| `self` | `sel-016-intimate`～`sel-020-intimate` | 仍以自我溫柔、脆弱面、自我接納與真心話為主；未轉成伴侶間身體親密協商。 |

## 7. US3 分類乾跑紀錄（2026-06-10）

以下以代表性候選題目對照 §4 分類清單做一次乾跑，包含五個主題與刻意模糊案例，確認維護者在**不依賴額外 implementation context** 的情況下，仍可依「主要對話任務」得到單一落點。

| 題目摘要 | 預期主題 | 判斷依據 |
|----------|----------|----------|
| 你什麼時候最容易因為我的一個小動作而心動？ | `attraction` | 核心是心動與吸引，對應 CB-001 / US3-1。 |
| 你覺得我哪種表情最有魅力，會讓你想更靠近？ | `attraction` | 焦點是魅力與浪漫注目，不是身體協商。 |
| 當我感到害羞時，我通常是在保護什麼？ | `self` | 主要任務是自我覺察與情緒理解，對應 CB-002 / US3-2。 |
| 我現在對親密這件事最常出現的猶豫或界線感是什麼？ | `self` | 雖提到親密，但任務是理解自己的界線，不是和伴侶協商。 |
| 我們最近最容易因為哪種訊息語氣而誤解彼此？ | `interaction` | 是日常溝通與互動節奏問題，對應 CB-003。 |
| 當其中一方需要獨處時，我們怎麼約定提醒彼此比較不會受傷？ | `interaction` | 核心是相處節奏與溝通安排，不是信任修復或身體親密。 |
| 當我受傷或退縮時，你做什麼最能讓我重新感到安全？ | `trust` | 主體是安全感與修復，對應 CB-004 / US3-4。 |
| 如果我們對未來承諾的步調不同，你最需要什麼保證才安心？ | `trust` | 核心是承諾與可靠感，而非成人身體親密。 |
| 我們要怎麼開口邀請身體親密，才會讓彼此都覺得被尊重？ | `desire` | 是成人身體親密中的邀請與同意協商，對應 US3-3。 |
| 如果今天不想更靠近，你希望我怎麼理解你的拒絕才不會有壓力？ | `desire` | 主體是身體親密中的拒絕、界線與安全，屬 desire。 |
| 當我很想靠近你時，我更希望先說感受，還是先問你現在的狀態？ | `desire` | 雖有情感語氣，但主要任務是進入身體親密前的邀請與確認。 |
| 重新建立安全感時，你會希望我們先抱抱，還是先把界線說清楚？ | `desire` | 刻意模糊案例；因重點落在成人身體親密中的界線協商，依 §4 第二條歸 `desire`。 |

**乾跑結果摘要**：
- 樣本數：12 題（`attraction` 2、`self` 2、`interaction` 2、`trust` 2、`desire` 4；其中 2 題為刻意模糊案例）
- 可在不查閱額外實作脈絡下直接唯一歸類：12 / 12（100%）
- 結論：已達成 SC-002 所需的「至少 90% 可分類」目標；後續新增卡牌可先依 §4 清單判斷，再進行內容審查。

## 9. US4 泰文（th）翻譯品質關卡（2026-06-10，直球改版）

依 R-005 與 Edge Case，逐張比對 20 張 desire 卡的 `th` 與權威語系 `zh`/`en`，確認泰譯**對齊 zh/en 的直球尺度**，保留「成人、同意、安全、可拒絕」語氣，且**不因語言轉換更露骨或更含糊**。

- **尺度對齊（不超譯）**：zh/en 直接用「做愛／性」時，th 以一般成人用語 `เซ็กส์`／`มีเซ็กส์`（sex／做愛）對應，未使用更粗俗或更露骨的詞；「性幻想」譯 `จินตนาการทางเพศ`（des-014）、「性需求」譯 `ความต้องการทางเพศ`（des-007）、「安全詞」譯 `เซฟเวิร์ด`（des-018），皆為與來源同尺度的直譯，未自行加碼描寫性行為過程。
- **同意／安全語氣保留**：關鍵語意詞皆有對應——`ปลอดภัย`（安全）、`เคารพ`（尊重）、`ขอบเขต`（界線）、`ขอหยุด`（可暫停／喊停，des-006/des-014）、`ปฏิเสธ`（可拒絕，des-017）、`ไม่บังคับ`／`ไม่กดดัน`／`หน้าที่`（非強迫／無壓力／義務語境，des-009/des-010）。
- **非強迫揭露**：th 沿用來源的開放問句結構（如 `มี…ไหม`、`คุณอยากให้…แบบไหน`），可回答「沒有」或選擇不揭露，未轉成測驗式或強制語氣。
- **結論**：20 / 20 th 文案通過品質關卡，尺度與 zh/en 一致；`ja` 依 R-005 為 `en` 佔位（值等於 en），四語系 schema 滿足。
