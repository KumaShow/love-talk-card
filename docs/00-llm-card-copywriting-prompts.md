# Love Talk Card 文案生成提示詞手冊

> 提供給 Gemini、ChatGPT 與其他 Web LLM 使用的卡片文案生成提示詞。
> 目標是先穩定產出高品質 `zh-TW` 卡片文案，再進行審稿、翻譯與 JSON 整理。

---

## 使用目標

這份文件用來協助你為 `src/data/cards.json` 規劃與生成卡片文案，重點不是一次要求 LLM 直接產出 80 張完整正式稿，而是透過分階段提示詞提高品質與可控性。

建議流程：

1. 先定義四個主題的文案邊界與遞進策略
2. 再逐一生成單一主題卡片
3. 生成後做主題內去重與修文
4. 最後做跨主題邊界校正
5. 確認 `zh-TW` 後，再翻譯 `en`
6. 最後才整理成 JSON

---

## 專案文案規格

請在所有提示詞中反覆強調以下規則：

- 專案定位：給情侶／伴侶使用的對話卡遊戲
- 使用情境：在手機上抽卡、輪流唸題、自然聊天
- 主語言：繁體中文 `zh-TW`
- 語氣：溫柔、自然、真誠、有安全感
- 不要寫成心理測驗、諮商問卷、企業訪談、雞湯語錄
- 題目要能被直接唸出口，不拗口
- 每題盡量只問一件事，避免雙重提問
- 要有題型變化，避免不同卡片只是同義改寫
- `intimate` 可以更私密，但不要露骨、不要羞辱、不要逼答
- 不強迫揭露創傷，不製造道德壓力
- 每題理想上可引導約 1 到 3 分鐘對話

資料結構與配比：

- 4 個主題：`attraction`、`self`、`interaction`、`trust`
- 每主題 20 張
- 每主題 `15 base + 5 intimate`
- `base` 建議配比：`level 1 = 5`、`level 2 = 6`、`level 3 = 4`
- `intimate` 建議配比：`level 2 = 2`、`level 3 = 3`

---

## 主題邊界摘要

### `attraction`｜心動瞬間

- 核心：吸引、心跳感、曖昧感、被喜歡與被看見的瞬間
- 適合：初見印象、持續心動、迷人細節、浪漫回憶、現在仍會被吸引的時刻
- 避免：承諾協商、未來藍圖、衝突修復、太多自我剖析

### `self`｜自我探索

- 核心：在關係中認識自己、看見需求、界線、脆弱與內在狀態
- 適合：個人特質、需求表達、內在矛盾、被理解的渴望、自我成長
- 避免：太偏互動技巧、太偏長期承諾、太偏曖昧心動

### `interaction`｜互動理解

- 核心：彼此如何溝通、傾聽、回應、誤解與修復
- 適合：溝通方式、衝突時反應、道歉與修復、被聽懂的感受、互動節奏
- 避免：過度個人內在探索、未來承諾主題、純浪漫回憶

### `trust`｜信任成長

- 核心：安全感、承諾、依靠、共同未來、深層坦白與關係成長
- 適合：信任如何建立、未來想像、承諾意義、脆弱與依靠、深層真話
- 避免：太多初期心動、太多單純溝通技巧、太多泛自我探索

## 推薦操作順序

最推薦的實作方式：

1. 先使用「主題規格規劃 Prompt」
2. 再依主題分別使用單一主題生成 Prompt
3. 每生成一組就立刻跑「主題內去重與修文 Prompt」
4. 四組都完成後，再跑「跨主題邊界校正 Prompt」
5. 最後再用「英文翻譯 Prompt」與「JSON 整理 Prompt」

不建議一開始就要求：

- 一次生成四個主題全部 80 張
- 一次生成四語版本
- 一次直接輸出最終 JSON 正式稿

## Prompt 1：主題規格規劃

用途：先讓 LLM 定義文案策略與邊界，不直接生成卡片。

```text
你現在是資深繁體中文對話卡牌編輯，正在協助設計一款給情侶／伴侶使用的對話卡遊戲。

請先不要直接生成卡片內容，先幫我規劃四個主題的文案策略與邊界。主題如下：

1. attraction｜心動瞬間
2. self｜自我探索
3. interaction｜互動理解
4. trust｜信任成長

請根據以下產品定位規劃：
- 使用情境：兩人在手機上輪流抽卡、唸題、聊天
- 語氣目標：溫柔、自然、真誠、有安全感、可被大多數伴侶接受
- 不要寫得像心理測驗、諮商問卷、企業訪談或社群語錄
- 要能被直接唸出口，不拗口
- intimate 題可以更私密，但不要露骨、不要冒犯、不要強迫揭露創傷
- 每張卡要能引導 1 到 3 分鐘的真實對話
- 主題邊界要明確，避免 attraction 跟 trust、self 跟 interaction 混在一起

請輸出：
A. 每個主題最核心的情緒與對話目的
B. 每個主題適合問的內容類型
C. 每個主題不適合問的內容類型
D. 四個主題彼此最容易混淆的地方
E. 每個主題的 level 1 / level 2 / level 3 應該如何遞進
F. intimate 題與 base 題的差別原則

請用條列、清楚分段、繁體中文輸出。
```

用途：一次只生成一個主題，品質最穩。

```text
你現在是繁體中文戀愛對話卡牌編輯。請為主題「{主題名稱} / {themeId}」生成卡片文案。

這個主題的定義：
{貼上主題專用定義段}

請生成共 20 張卡：
- 15 張 base
- 5 張 intimate

配比要求：
- base：level 1 = 5 張、level 2 = 6 張、level 3 = 4 張
- intimate：level 2 = 2 張、level 3 = 3 張

文案要求：
- 只產出繁體中文 `zh-TW`
- 每題一句為主，必要時可兩小句，但不要冗長
- 每題只問一件事，避免雙重提問
- 要自然口語、適合伴侶直接念出來
- 避免過度抽象，例如「你如何定義愛」這種太大題
- 避免和其他主題混題
- intimate 可以帶有身體親近、渴望、私密感，但不要露骨
- 題型要分散：回憶型、感受型、觀察型、現在式、未來期待型
- 避免同一個切角重複太多次

請用表格輸出，欄位為：
id_prefix | level | isIntimate | 文案 | 設計理由

其中：
- id_prefix 先用 att-001 / sel-001 / int-001 / tru-001 這種格式即可
- isIntimate 請填 true / false
- 設計理由只要一句，說明這題為什麼屬於這個主題
```

## Prompt 3：主題專用定義段

以下段落可直接替換到「單一主題生成模板」中的 `{貼上主題專用定義段}`。

### `attraction`｜心動瞬間

```text
- 聚焦在彼此吸引、初期心動、持續被吸引的瞬間、曖昧與心跳感
- 不是在談長期承諾、修復衝突、價值觀協商
- 要有浪漫感，但不能太浮誇、太偶像劇
- 要讓人感覺「這題很適合今晚聊」
- 不要過度集中在「第一次見面」一種角度，要包含現在式的心動與持續吸引
```

### `self`｜自我探索

```text
- 聚焦在關係中的自我認識、自我需求、界線、脆弱、成長與內在狀態
- 重點是「我怎麼理解自己」，不是「我們怎麼溝通」
- 不是在談衝突修復技巧，也不是在談未來承諾
- 要讓對方有機會看見自己平常不容易說出口的內在
- 題目要讓人更認識自己，而不只是描述對方怎麼對待自己
```

### `interaction`｜互動理解

```text
- 聚焦在彼此怎麼表達、接收、傾聽、誤解、修復、回應需求
- 重點是「我們怎麼互動」，不是單純談個人內在
- 可以碰到不安與誤會，但重點要落在互動方式與理解橋樑
- 不要寫成太多人生觀或未來藍圖，那比較像 trust
- 題目應該幫助雙方更知道彼此怎麼被聽見、怎麼被接住
```

### `trust`｜信任成長

```text
- 聚焦在信任、承諾、安全感、共同未來、深層坦白與關係成長
- 重點是「我們要怎麼成為更穩固的彼此」
- 可以談脆弱、依靠、害怕失去、未來想像
- 不要太偏曖昧心動，也不要太偏單純溝通技巧
- 題目應該帶出更深層的安全感與長期關係想像
```

## Prompt 4：主題內去重與修文

用途：審查同一主題內部是否重複、跑題或語感不自然。

```text
請你扮演嚴格的繁體中文卡牌文案總編輯，檢查我下面這一組卡片。

請從以下角度審查並重寫：
1. 是否有重複題型或重複切角
2. 是否有雙重提問、句子過長、口語不自然
3. 是否有太抽象、太空泛、太像心理測驗
4. 是否有主題跑偏
5. level 遞進是否自然
6. intimate 是否夠私密但不冒犯

請輸出三個區塊：
A. 整體評語
B. 建議刪除或重寫的題目清單
C. 重寫後的完整版本

重寫原則：
- 保留原本主題核心
- 讓語氣更自然、溫柔、可直接念
- 讓 20 張卡之間的角度更分散
- 仍然只輸出繁體中文
```

## Prompt 5：跨主題邊界校正

用途：檢查四個主題之間是否互相混題。

```text
我已經有四組主題卡片草稿：attraction、self、interaction、trust。

請你擔任內容架構編輯，做「跨主題邊界校正」。

檢查目標：
- attraction 是否太常寫成 trust 或 self
- self 是否太常寫成 interaction
- interaction 是否太常寫成 trust
- trust 是否太常寫成 attraction 或 self

請輸出：
1. 每個主題最清楚的代表特徵
2. 每個主題最常混到哪一個主題
3. 可疑卡片清單：指出是哪一張、為什麼跑題
4. 建議改寫版本
5. 若有兩張其實在問同一件事，也請指出

原則：
- 不要重寫全部，只抓最需要修的題目
- 以繁體中文回答
- 評語要具體，不要空泛
```

## Prompt 6：英文翻譯

用途：在 `zh-TW` 確認後，再翻成 `en`。

```text
請將下面這批戀愛對話卡的繁體中文文案翻譯成自然英文。

翻譯原則：
- 保留溫柔、自然、可直接唸出口的口語感
- 不要直譯成僵硬英文
- 不要翻得太文青或太戲劇化
- intimate 題要保留親密與安全感，但避免過度露骨
- 每題仍應像是伴侶可以直接對彼此說出口的問題
- 如果中文原句稍微不順，你可以做最小幅度的英文意譯，但不要改變核心意思

請輸出表格，欄位為：
id | zh-TW | English
```

## Prompt 7：JSON 整理

用途：將已確認的文案轉為專案資料格式。

```text
請將我確認過的卡片文案整理成 JSON 陣列，每個物件格式如下：

{
  "id": "att-001-base",
  "theme": "attraction",
  "isIntimate": false,
  "level": 1,
  "text": {
    "zh": "繁體中文文案",
    "en": "英文翻譯"
  }
}

規則：
- theme 只能是 attraction / self / interaction / trust
- isIntimate 為 true 時，id 結尾要用 intimate；否則用 base
- base 與 intimate 的數量要正確
- level 要保留原本設定
- zh 必須是繁體中文
- en 要自然，不要直譯得很生硬
- 先不要輸出 th / ja
- 只輸出合法 JSON
- 不要多加說明文字
- 不要使用 markdown code block
```

3. 生成後立刻把結果丟給 Prompt 4 做修文
4. 四個主題都完成後，用 Prompt 5 做跨主題校正
5. 最後才做英文翻譯與 JSON 整理

如果你發現某個模型容易失控，可以加上這句：

```text
如果你發現某些題目彼此太像，請主動刪除重複題型並補上不同角度，不要只做同義改寫。
```

如果你發現模型常常跑題，可以加上這句：

```text
在輸出前，請先自行檢查每題是否真的屬於該主題；若更像其他主題，請直接改寫後再輸出。
```

---

## 編輯建議

最後人工審稿時，優先檢查這幾件事：

- 是否有太多題目都在問同一件事
- 是否有些題目比較像別的主題
- `level 1` 是否真的夠好聊、夠容易開口
- `level 3` 是否有深度，但不造成壓力
- `intimate` 是否有私密感，但不會讓多數使用者立刻退縮
- 文案是否真的像伴侶之間會說出口的句子

---

## 補充建議

- 先完成 `zh-TW`，不要一開始就四語一起做
- 一次只做一個主題，品質通常明顯更穩
- 先要求 LLM 輸出「設計理由」，有助於抓出跑題卡
- 可以交叉使用不同模型：一個負責生成，另一個負責審稿
- 若要正式進版，建議最後仍由人工做一次整體語氣統一
