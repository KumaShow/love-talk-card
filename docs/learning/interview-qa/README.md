# 面試問答章節

把 `typescript/` 與 `testing/` 兩章學到的觀念，轉成「問題 → 答案」的卡片式整理。

## 為什麼另開一章

- 主題筆記的目的是「理解」，面試 Q&A 的目的是「輸出」。
- 同一個觀念，讀懂 ≠ 能用 60 秒講清楚。這章逼你練輸出。
- 面試時可快速翻找自己寫過的答案，不用每次重新想怎麼回答。

## 檔案

- [`typescript.md`](./typescript.md) — TS 相關面試題
- [`testing.md`](./testing.md) — 前端測試相關
- [`vue-architecture.md`](./vue-architecture.md) — Vue 3 / Pinia / Composition API / Router 架構題

## 單題模板

每題都用下面的格式寫（主題筆記有類似結構，但面試版更濃縮）：

```markdown
### Q: {{問題}}

**考點**：面試官想聽什麼（1 行）

**30 秒版答案**：
- 結論先講
- 2–3 個支撐點
- 舉 1 個本專案的例子（`file:line`）

**2 分鐘版答案**：
（擴充到能講的所有細節，含對比、trade-off）

**可能追問**：
- 追問 1：……
- 追問 2：……

**我答不出的時候**：
（誠實記：這題現在答不好，需要再讀哪裡補。）
```

## 使用方式

- **每週挑 2–3 題練口述**：對自己、對鏡子、對朋友都可以。
- **30 秒 / 2 分鐘兩版**：面試通常先簡答，追問才展開；兩版都練。
- **引用 `file:line`**：具體例子是跟別人差異化的關鍵。
- **定期更新「我答不出的時候」**：這欄最誠實。一段時間後再回來看會很有成就感（或很羞恥，都是進步）。

## 維護節奏

- 每讀完一個主題筆記 → 補 2–3 題面試問答。
- 真的面試遇到沒答好的題 → 立刻回來補答案。
- 別一次寫太多；寫 5 題紮實的，勝過寫 30 題每題三行。
