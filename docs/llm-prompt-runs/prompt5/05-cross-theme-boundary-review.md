# 跨主題邊界校正審查

來源草稿：

- `docs/llm-prompt-runs/draft/attraction-draft.md`
- `docs/llm-prompt-runs/draft/self-draft.md`
- `docs/llm-prompt-runs/draft/interaction-draft.md`
- `docs/llm-prompt-runs/draft/trust-draft.md`

## 整體結論

- 四組草稿整體主題骨架清楚，`attraction` 最穩定。
- 主要混題集中在 `interaction` 與 `trust`，因為兩者都容易碰到安撫、誤會、修復。
- `self` 整體方向正確，但少數題目把焦點滑到「怎麼互動」而不是「怎麼理解自己」。
- 本輪不需要全面重寫，只需優先修正 7 張高風險卡片。

## 1. 每個主題最清楚的代表特徵

### attraction

- 聚焦心動起點與持續被吸引的瞬間。
- 適合寫迷人的小動作、靠近時的感覺、日常中的魅力。
- 核心語氣是曖昧、浪漫、被吸引，而不是安全感或承諾。

### self

- 聚焦在關係裡如何認識自己。
- 適合寫情緒模式、需求、界線、自我形象與內在矛盾。
- 核心焦點是「我怎麼理解自己」，不是「我們怎麼協調」。

### interaction

- 聚焦兩人如何表達、接收、誤會、修復與重新對上頻率。
- 適合寫傾聽方式、說話風格、衝突後怎麼回到日常。
- 核心焦點是互動機制，不是長期安全感。

### trust

- 聚焦安全感、依靠、誠實、共同承擔與未來感。
- 適合寫什麼會讓人更敢交付脆弱、什麼會讓人更放心。
- 核心焦點是關係穩定度與深層可靠感，不是單次溝通技巧。

## 2. 每個主題最常混到哪一個主題

| 主題 | 最常混到 | 具體原因 |
|---|---|---|
| attraction | trust | 一旦題目從「心動／魅力」轉成「在意程度／被安穩接住」，就會變成信任感。 |
| self | interaction | 一旦開始問「你怎麼反應、怎麼開口」，焦點就會從自我認知滑向互動表現。 |
| interaction | trust | 一旦重點變成「安心、被接住、被放在心上」，就更像安全感而非互動方式。 |
| trust | self | 一旦只剩「我有哪些脆弱」，沒有回到「你怎麼讓我更能信任你」，就會偏自我探索。 |

## 4. 建議改寫版本

| 卡片 | 原文 | 建議改寫 | 改寫說明 |
|---|---|---|---|
| `att-009` | 有沒有哪個「瞬間」，讓你突然發現自己比想像中還在意我？ | 有沒有哪個瞬間，讓你突然覺得我比原本更有魅力？ | 把重點從依附感拉回吸引力提升。 |
| `sel-007` | 你生氣的第一瞬間，通常會先怎麼反應？ | 你一生氣時，心裡最先冒出來的感覺通常是什麼？ | 從外顯反應改成內在辨識。 |
| `sel-019` | 你有沒有一個很想被照顧，卻不太會開口的時候？ | 有沒有哪種時候，你其實很想被照顧，卻會先把這個需要藏起來？ | 保留脆弱感，但把焦點放回自我需求隱藏。 |
| `int-016` | 當你難過的時候，我怎麼做會讓你覺得被接住？ | 當你難過時，我先做哪一件事，會讓你比較願意跟我說下去？ | 從安全感改成互動入口。 |
| `int-020` | 哪一個小動作，會讓你覺得自己又被我放在心上？ | 我做哪一個小動作時，你最容易感覺到我有把你的話聽進去？ | 從被珍惜改成被理解。 |
| `tru-011` | 如果我們之間出現誤會，你希望我第一步做什麼？ | 如果我們之間出現誤會，你最需要我守住什麼，才不會動搖你對我的信任？ | 把流程題改成信任條件題。 |
| `tru-016` | 你有沒有一個比較脆弱的地方，希望我能多溫柔一點對待？ | 有沒有一個你比較脆弱的地方，是你希望我知道後，能被我穩穩接住的？ | 把脆弱和可靠承接綁回 trust。 |

## 建議優先順序

1. 先修 `tru-011`，因為它和 `interaction` 的重疊最明顯。
2. 再修 `int-016`、`int-020`，把 `interaction` 從安全感語彙拉回互動語彙。
3. 接著修 `sel-007`、`sel-019`，把焦點從反應方式拉回自我辨識。
4. 最後修 `att-009`、`tru-016`，補強主題純度。
