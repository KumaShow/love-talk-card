# Trust｜信任成長 Prompt

你現在是繁體中文戀愛對話卡牌編輯。請為主題「信任成長 / trust」生成卡片文案。

這個主題的定義：
- 核心情緒與對話目的：
  - 核心情緒：安穩、承諾、歸屬。
  - 對話目的：處理「脆弱與不確定性」，建立深度安全感與長期共識。
  - 主題重點：重點是「我們要怎麼成為更穩固的彼此」，不是單純聊心動感。
- 適合問的內容類型：
  - 對承諾的定義。
  - 對未來的共同想像。
  - 在對方面前展現脆弱的感受。
  - 如何處理嫉妒或不安。
  - 支持系統的建構。
  - 願意一起承擔什麼。
  - 關係裡最重要的原則。
- 不適合問的內容類型：
  - 嚴禁「忠誠度測試」（如：如果你媽跟我掉進水裡）或帶有威脅感的假設。
  - 單純的曖昧感。
  - 初次心動。
  - 外表吸引力。
  - 日常小習慣偏好。
  - 這些太輕，屬於 attraction 或 interaction。
- intimate 與 base 的差別：
  - Base 題（基礎題）：
    - 屬性：溫和、安定，適合在放鬆且能好好說話的時刻進行（例如晚餐後、安靜散步時）。
    - 對話目標：建立基本的信任語言，讓安全感、依靠與承諾類話題能自然展開。
  - Intimate 題（私密題）：
    - 屬性：高感性、高脆弱性、適合安靜的私人空間（睡前、居家）。
    - 差別原則：
      - 深度不同：Intimate 題會要求更多「情緒揭露」，而不只是「資訊分享」。
      - 安全感焦點：可以談依靠、信任推進、害怕失去與長期承接，但不要變成忠誠測試。
      - 優先權：強調「我們如何成為更穩固的彼此」。

請生成共 20 張卡：
- 15 張 base
- 5 張 intimate

配比要求：
- base：level 1 = 5 張、level 2 = 6 張、level 3 = 4 張
- intimate：level 2 = 2 張、level 3 = 3 張

文案要求：
- 只產出繁體中文 `zh-TW`
- 每題一句為主，必要時可兩小句，但不要冗長
- 每題只問一件事，避免雙重提問
- 要自然口語、適合伴侶直接念出來
- 避免過度抽象，例如「你如何定義愛」這種太大題
- intimate 卡請依照上方「差別原則」撰寫，保持私密但不要露骨
- 題型要分散：安全感、承諾、依靠、真話、未來成長
- 避免同一個切角重複太多次
- 避免和其他主題混題：
  - **Trust vs. Attraction**
    - 混淆點：都涉及對另一半的感受。
    - 區分：Attraction 是「我看著你時眼裡的火花」；Trust 是「我背對著你時心裡的安穩」。
    - ❌「你最喜歡我哪裡？」
      這是 attraction。
    - ✅「什麼會讓你對一段關係失去信任？」
      這是 trust。
- Level 遞進：
  - Level 1｜安全感認知
    - 怎樣會讓你安心。
    - 什麼行為會讓你失望。
    - 你最重視的關係原則。
    - 從安全感與基本原則切入，讓人容易回答。
  - Level 2｜承諾與脆弱
    - 你最怕被辜負的是什麼。
    - 被信任對你來說意味著什麼。
    - 你需要多久才會真正相信一個人。
    - 開始觸及依靠、信任建立與脆弱感。
  - Level 3｜深層承接感
    - 如果你很脆弱，希望我怎麼接住你。
    - 哪件事會讓你覺得我們真的能走很久。
    - 你最希望我永遠不要放棄的是什麼。
    - 更靠近長期承諾、深層依靠與被承接的感受。

  - 範例遞進表：

    | 主題 | Level 1 | Level 2 | Level 3 |
    | :--- | :--- | :--- | :--- |
    | **Trust** | 你覺得我是個守信用的人嗎？ | 在什麼情況下，你會覺得「有你在真好」？ | 當你感到最脆弱時，你相信我會如何接住你嗎？ |
