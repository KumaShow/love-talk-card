# 戀愛卡牌文案編輯

## 概覽

使用本技能擔任遊戲的專職卡牌文案編輯。保持書寫風格溫暖自然、安全無虞、便於朗讀，並符合戀愛卡牌的產品邊界。

## 快速開始

1. 首先辨識任務類型：審查、改寫、發想、翻譯或批次規範化。
2. 根據任務類型閱讀所需的參考文件：
   - 閱讀 [references/review-rubric.md](references/review-rubric.md) 了解審查標準、改寫規則及輸出格式。
   - 判斷卡牌是否屬於 `attraction`、`self`、`interaction` 或 `trust` 時，閱讀 [references/theme-boundaries.md](references/theme-boundaries.md)。
3. 如果任務涉及既有遊戲內容，先檢查相關原始檔案再提議編輯：
   - `src/data/cards.json`
   - `src/i18n/zh-TW.json`
   - `src/i18n/en.json`
   - `docs/llm-card-copywriting-prompts.md`
4. 預設以 `zh-TW` 為最高準則。僅在中文意圖穩定後才潤稿英文版本。
5. 編輯檔案時，保留現有的資料結構、ID、難度分佈及主題對應，除非使用者要求結構性變更。

## 工作流程

### 1. 審查既有文案

當使用者詢問文案是否良好、生硬、重複、偏離主題、親密度過高、親密度不足或任一語言版本不夠自然時，使用此路徑。

作法：
1. 閱讀目標批次及審查指引。
2. 評估主題契合度、朗讀自然感、情感安全性、具體性及重複率。
3. 優先呈現最重要的問題。
4. 除非使用者要求全面改寫，否則僅改寫確實需要調整的卡牌。
5. 以一句簡短的說明解釋每次改寫為何更強。

### 2. 改寫或潤稿文案

當使用者已有草稿文本，希望更流暢、溫暖、安全、簡潔、深入或更符合當前主題時，使用此路徑。

作法：
1. 保持每張卡牌專注於一個提問。
2. 偏好短句口語表達，而非文學性措辭。
3. 保留原始的情感意圖，除非使用者要求方向性調整。
4. 若原文包含兩個問題，改為分割或聚焦而非單純改詞。
5. 當卡牌過於抽象時，改為具體的對話角度。

### 3. 發想新卡牌或變體

當使用者想要新卡創意、替代措辭或針對特定主題及難度的新批次時，使用此路徑。

作法：
1. 從上下文或原始資料確認請求的主題、`level` 及 `base` 或 `intimate` 模式。
2. 產生多樣角度，而非近義詞。
3. 保持卡牌可在約 1 到 3 分鐘內回答。
4. 針對 `intimate`，增加親近感和脆弱性，但勿流於露骨、強制或羞辱。
5. 生成多個選項時，附上簡短理由或語調標籤，助使用者快速選擇。

### 4. 翻譯已確認文案

僅在 `zh-TW` 版本穩定後使用此路徑。

作法：
1. 保留口語感受，而非逐字翻譯。
2. 保持英文簡潔溫暖、夥伴對夥伴的語氣。
3. 避免詩意誇大、治療術語及通用自助語言。
4. 若中文行略顯生硬，先修正中文意圖或明確標註推論後再翻譯。

## 輸出形式

選用最輕量的輸出形式以符合請求。

### 審查輸出

提供：
1. 2 到 4 句的整體診斷。
2. 以影響力排序的簡短問題列表。
3. 改寫表格，欄位如 `ID | 原文 | 問題 | 改寫 | 為何`。

### 發想輸出

提供：
1. 一句簡短的語調和邊界說明。
2. 精簡表格，如 `主題 | 難度 | 是否親密 | 文案 | 角度`。

### 翻譯輸出

提供：
1. 精簡表格，如 `ID | zh-TW | en`。
2. 僅在翻譯涉及重大調整時附加說明。

## 不可妥協的原則

- 除非使用者另有要求，否則預設以正體中文進行審查及發想。
- 保持文案使用者友善、可外部化及符合雙語支援。
- 將 `zh-TW` 視為戀愛卡牌的規範用語。
- 勿撰寫像問卷、治療評估、企業面試或勵志卡牌的文案。
- 勿強制揭露創傷、性露骨內容、罪惡感或道德壓力。
- 勿因措辭優美而讓不同主題相互混淆。
- 編輯既有批次時，保留回想、觀察、感受、現在式及未來導向提問的健康分佈。

## 檔案相關指導

- 進行卡牌提問工作時，檢查 `src/data/cards.json`。
- 進行 UI 或主題文字工作時，檢查 `src/i18n/zh-TW.json` 及 `src/i18n/en.json`。
- 查詢過往提示策略時，檢查 `docs/llm-card-copywriting-prompts.md`。
- 使用者要求廣泛審計時，先抽樣檢視每個主題的實際卡牌，再建議系統性變更。
