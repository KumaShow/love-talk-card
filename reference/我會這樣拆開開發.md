建議的技術設計

 1. 前端：Vue 3 + Vite + TypeScript  
 2. 狀態管理：prototype 階段用 Composition API + ref/computed 就夠，不必先上 Pinia  
 3. 資料格式：每張卡固定 schema，例如 id, theme, isIntimate, text.zh, text.en, 
text.th  
 4. 路由/頁面：首頁、抽卡頁兩頁即可  
 5. 動畫：先用 CSS 3D flip，避免早期引入過重動畫庫  
 6. 抽卡邏輯：先做單次 session 內不重複；持久化記憶放到下一個 change

建議的 Spectra 搭配方式

因為你現在還在收斂方案，這幾個指令最適合：

 1. 先正式立案 /spectra:propose love-talk-prototype
 
 這個 change 建議只包含：
  - Vue + Vite + TS 專案骨架
  - 首頁主題選擇
  - 抽卡/翻牌流程
  - 假資料題庫
  - zh/en 顯示與 th schema 預留
 2. 如果要把 UI/資料結構先討論清楚 /spectra:discuss love-talk prototype architecture
 
 適合討論：
  - 卡片 JSON schema
  - 首頁與抽卡頁資訊架構
  - 多語排版比例
  - 私密模式是否在 prototype 就顯示入口
 3. 如果你想從規格反查現況 /spectra:ask love-talk prototype scope
 
 適合用來問：
  - 某個需求目前是否已在 spec 裡
  - 哪些功能應該屬於同一個 change
  - 某個設計決策應寫進 proposal.md 還是 design.md
 4. 確定規格後開始實作 /spectra:apply love-talk-prototype
 
 這時才離開 discuss，進入真正開發。
 5. 做完後補安全與品質檢查 /spectra:audit love-talk-prototype
 6. 完成並合併後封存 /spectra:archive love-talk-prototype
